# Learning & Thinking Log

这份日志记录我的理解怎样从“记住算法步骤”逐渐转向“用统一问题框架解释算法”。

## 阶段 1：先理解价值函数为什么存在

最初我把 state value 当作一个抽象评分。后来我意识到，它的核心作用是把一个状态之后无数条可能轨迹压缩成期望回报，使长期决策可以在当前状态上进行比较。

关键转变：

- reward 是一步反馈，return 是未来奖励的累计；
- `Vπ(s)` 是按策略平均后的状态价值；
- `Qπ(s,a)` 是先固定动作，再按策略行动的价值；
- `Vπ(s)=Σₐπ(a|s)Qπ(s,a)`，而不是 `V=Q`。

## 阶段 2：从 Bellman 方程理解“递推”

我曾困惑：既然 Bellman 最优方程写成 `v=f(v)`，为什么算法又写成 `vₖ₊₁=f(vₖ)`？

现在我的理解是：

- 固定点方程定义答案；
- 迭代方程定义求解过程；
- `γ<1` 带来的收缩性解释了为什么反复更新能接近唯一固定点。

## 阶段 3：把 DP、MC、TD 放在同一坐标系

三者不只是三种算法，而是三种获得学习目标的方式：

| 方法 | 信息来源 | target | 代价 |
|---|---|---|---|
| DP | 已知环境模型 | 精确一步期望 | 需要模型与状态遍历 |
| MC | 完整采样回合 | 实际 return | 等待终止，方差较大 |
| TD | 一步样本 + 当前估计 | reward + bootstrap | 有估计偏差，但可在线更新 |

这也让我更容易理解 GAE：它不是凭空出现，而是在“一步 TD—完整回报”连续谱上选择折中。

## 阶段 4：从表格方法走向深度强化学习

函数逼近解决了状态空间太大的问题，同时也引入了共享参数导致的耦合：一个样本会改变许多状态的估计。DQN 的经验回放与目标网络，本质上分别在处理样本相关性和移动目标。

我的工程判断因此变成：遇到训练震荡时，不能只调 learning rate，还要检查是否同时存在 off-policy、bootstrapping 与 function approximation 的组合风险。

## 阶段 5：从策略梯度走向 Actor–Critic

策略梯度让我第一次把“策略”看作可直接优化的分布，而不只是价值函数的贪心附属品。baseline 把绝对回报变成相对表现，Actor–Critic 再让 critic 学习这个 baseline。

关键连接：

`E[TD error | s,a] = Advantage(s,a)`（当价值估计准确时）

这解释了 actor 和 critic 为什么能共享学习信号。

## 阶段 6：把 PPO 参数翻译回理论语言

现在阅读 PPO 配置时，我不再只记参数名，而会进行如下翻译：

- `gamma`：未来奖励的时间尺度；
- `lambda`：GAE 的偏差—方差折中；
- `clip_param`：允许单轮策略改变的幅度；
- `num_envs × rollout_length`：一次更新的数据分布和批量；
- `reward weights`：任务目标本身的定义；
- `termination`：回合边界与 bootstrap 语义。
