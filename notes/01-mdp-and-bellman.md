# 01 · MDP, Value Functions and Bellman Equations

## MDP 在解决什么

MDP 把现实任务写成：

`M = (S, A, P, R, γ)`

其中最容易被低估的是状态 `S`：状态必须包含预测下一步所需的信息。对机器人任务而言，观测设计不只是网络输入工程，而是“我是否把问题建成了近似马尔可夫过程”。

## Reward 与 Return

`Gₜ = Rₜ₊₁ + γRₜ₊₂ + γ²Rₜ₊₃ + ...`

- reward：一步反馈；
- return：策略真正优化的长期成绩；
- `γ`：未来奖励的时间尺度，而不是学习率。

## V 与 Q

`Vπ(s) = E[Gₜ | Sₜ=s]`

`Qπ(s,a) = E[Gₜ | Sₜ=s, Aₜ=a]`

关键关系：

`Vπ(s) = Σₐ π(a|s) Qπ(s,a)`

所以 V 不是与每个 Q 相等，而是 Q 在策略动作分布下的平均。

## Bellman 方程怎样出现

先拆 return：

`Gₜ = Rₜ₊₁ + γGₜ₊₁`

再取条件期望：

`Vπ(s) = Eπ[Rₜ₊₁ + γVπ(Sₜ₊₁) | Sₜ=s]`

它把“整段未来”压缩成“一步奖励 + 下一状态价值”。这就是后续所有 bootstrap 方法的共同骨架。

## 为什么写成 v=f(v)

`v=f(v)` 表示价值函数是 Bellman 映射的固定点。它定义正确答案。

`vₖ₊₁=f(vₖ)` 表示从任意初值反复应用映射。它定义求解过程。

当 `γ<1` 时，Bellman 映射在最大范数下是收缩映射：

`||T(v)-T(u)||∞ ≤ γ||v-u||∞`

每次迭代都会缩小估计间的最大差距，因此可以逼近唯一固定点。

## 最优 Bellman 方程的 max

`V*(s) = maxₐ E[R + γV*(S') | s,a]`

`max` 返回最大价值，`argmax` 返回实现最大价值的动作。右侧的局部动作比较，隐含地定义了贪心最优策略。
